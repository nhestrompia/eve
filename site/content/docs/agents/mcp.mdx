---
title: MCP
description: Connect coding agents to local eve product history.
---

eve exposes local product history to agents through the Model Context Protocol. This lets an agent read snapshots, complete a new snapshot, or check out a recorded implementation state through structured tools.

## Transports

Stdio:

```sh
eve mcp-stdio
```

Streamable HTTP, after starting the local runtime:

```sh
eve dev
```

Connect the client to:

```text
http://localhost:4317/mcp
```

## Resources

```text
eve://repos
eve://repos/{repoId}
eve://repos/{repoId}/snapshots
eve://repos/{repoId}/snapshots/{snapshotId}
```

## Tools

- `list_repos`
- `list_snapshots`
- `get_snapshot`
- `pending_snapshot`
- `complete_snapshot`
- `skip_snapshot`
- `checkout_snapshot`

`skip_snapshot` requires a non-empty reason and writes a durable `.eve/skips/`
record. Skips resolve reviewed committed ranges without appearing in Snapshot
timelines, changelogs, counts, or search.

## Recommended setup

```sh
npx @nhestrompia/eve@latest install
```

This installs the platform-specific EVE binary and configures Codex, Claude
Code, and opencode with its absolute path. To reconfigure MCP after EVE is
installed:

```sh
eve install-mcp
```

## Security model

Version 0 assumes trusted local usage. Keep local HTTP bound to localhost and only connect agent clients you trust.
