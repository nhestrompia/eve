## Usage

```sh
eve <command>
```

## Commands

### init

Initialize eve storage in the current Git repository and install EVE's managed
agent guidance in `AGENTS.md` and `CLAUDE.md`.

```sh
eve init
eve init --no-agent-instructions
eve init --instructions-only
```

Existing content outside the EVE markers is preserved. `--instructions-only`
does not create or update `.eve/` state.

### instructions

Install, inspect, and compare the repository's managed EVE instruction blocks.

```sh
eve instructions install
eve instructions install --target agents
eve instructions install --target claude
eve instructions install --force
eve instructions status
eve instructions diff
```

Missing and exact older canonical blocks can be installed safely. A modified
block requires `--force`; malformed or duplicate markers must be repaired
manually. `status` returns a non-zero exit code unless both files are current.

### doctor

Check repository storage, MCP setup, agent instructions, required Snapshot
tools, and Snapshot activity.

```sh
eve doctor
```

Missing MCP configuration or a first Snapshot is reported as an onboarding
warning. Invalid repository state or non-current instruction files return a
non-zero exit code.

### add

Create or update a staged snapshot draft. Use this after implementation and verification, when you know what product result the snapshot should describe.

```sh
eve add --title <title> --summary <summary> [--type feature] [--validation <command>]
```

Common flags:

- `--title`
- `--type`
- `--summary`
- `--user-visible-change`
- `--decision`
- `--risk`
- `--timeline`
- `--validation`
- `--artifact`
- `--corrects`
- `--supersedes`
- `--reverts`
- `--depends-on`
- `--related`
- `--json`
- `--allow-dirty`

### commit

Validate the staged draft, derive Git facts, and write the canonical snapshot under `.eve/snapshots/`.

```sh
eve commit
eve commit --allow-dirty
```

### dev

Start the local runtime, product UI, local API, and HTTP MCP endpoint.

```sh
eve dev
eve dev --addr localhost:4317
```

### mcp-stdio

Start the MCP server over stdio.

```sh
eve mcp-stdio
eve mcp-stdio --cwd /path/to/repo
```

### install-mcp

Configure supported MCP clients.

```sh
eve install-mcp
eve install-mcp --install
eve install-mcp --clients codex,claude
```

### snapshot

Print a snapshot.

```sh
eve snapshot <snapshot-id>
```

### changelog

Generate release notes from completed snapshots instead of Git commits.

```sh
eve changelog
eve changelog --since <snapshot-id>
eve changelog --since 2026-07-01
eve changelog --from <snapshot-id> --to <snapshot-id>
eve changelog --repo-id <repo-id>
eve changelog --cwd /path/to/repo
eve changelog --markdown
```

`--since <snapshot-id>` excludes the boundary snapshot and includes later snapshots. Date-based ranges include snapshots on or after the provided UTC date.

### compare

Compare two product states using snapshot history.

```sh
eve compare <from-snapshot-id> <to-snapshot-id>
eve compare <from-snapshot-id> <to-snapshot-id> --markdown
eve compare --repo-id <repo-id> <from-snapshot-id> <to-snapshot-id>
eve compare --cwd /path/to/repo <from-snapshot-id> <to-snapshot-id>
```

Comparison output focuses on product changes, decisions, risks, validation, and timeline entries. It does not produce a Git diff.

### checkout

Check out the Git state associated with a snapshot.

```sh
eve checkout <snapshot-id>
eve checkout --force <snapshot-id>
```

### validate

Validate a snapshot JSON file.

```sh
eve validate .eve/snapshots/<snapshot-id>.json
```

### canonicalize

Write canonical JSON for a snapshot.

```sh
eve canonicalize .eve/snapshots/<snapshot-id>.json
```

### version

Print the CLI and snapshot schema versions.

```sh
eve version
```
