---
title: Review History
description: Inspect snapshots from the local UI and CLI.
---

eve provides a local web UI for reviewing repository product history without sending repository data to a hosted service.

```sh
eve dev
```

Open `http://localhost:4317`.

## What to review

Read snapshots the way you would read a product changelog with evidence attached:

- What changed?
- Why did it change?
- How was it verified?
- Which decisions and risks matter?
- Which artifacts prove the behavior?
- Which Git state implements it?
- Which snapshots are related?

## Inspect from the CLI

```sh
eve snapshot <snapshot-id>
```

## Checkout an implementation state

```sh
eve checkout <snapshot-id>
```

Use `--force` only when you intentionally want to overwrite local working tree state:

```sh
eve checkout --force <snapshot-id>
```

## Validate a snapshot file

```sh
eve validate .eve/snapshots/<snapshot-id>.json
```

## Canonicalize a snapshot file

```sh
eve canonicalize .eve/snapshots/<snapshot-id>.json
```
