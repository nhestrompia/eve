# RFC 0001: Snapshot-First EVE Runtime

## Decision

Use Snapshot everywhere.

Snapshot means a completed product unit. Commit means an implementation
checkpoint. Git state means the resolved repository state that implements the
Snapshot.

One feature, bug fix, experiment, refactor, or release creates one Snapshot.
Snapshots are task-scoped, not commit-scoped.

## Runtime

`eve dev` starts the local runtime:

```text
EVE Runtime
├── Web UI
├── Local API
├── MCP server
├── Repo watcher/cache refresh
└── Local index cache
```

The local DB/cache is rebuildable. Canonical state is stored in:

```text
<repo>/.eve/snapshots/*.json
```

## Snapshot Shape

```json
{
  "id": "snap_123",
  "schemaVersion": "0.1.0",
  "title": "Add GitHub OAuth",
  "type": "feature",
  "summary": "Users can now sign in with GitHub.",
  "userVisibleChange": "The login screen now includes a GitHub sign-in option.",
  "relationships": {
    "corrects": [],
    "supersedes": [],
    "reverts": [],
    "dependsOn": [],
    "related": []
  },
  "risks": [],
  "timeline": [],
  "decisions": [],
  "validation": [],
  "artifacts": [],
  "implementation": {
    "branch": "main",
    "gitState": "abc123",
    "commits": ["abc123"],
    "dirty": false
  },
  "createdAt": "2026-07-03T15:00:00Z"
}
```

Omitted optional arrays and relationship fields default to empty arrays when
canonicalized.

## Git Facts

Agents provide product meaning. EVE derives implementation facts:

- `branch`
- `gitState`
- `baseCommit`
- `commits`
- `dirty`

The runtime uses Git commands such as `git branch --show-current`,
`git rev-parse HEAD`, `git status --porcelain`, and `git log`.

## API and MCP

Local API:

```text
GET  /api/repos
GET  /api/repos/{repoId}
GET  /api/repos/{repoId}/snapshots
GET  /api/repos/{repoId}/snapshots/{snapshotId}
POST /api/repos/{repoId}/snapshots/{snapshotId}/checkout
```

MCP resources:

```text
eve://repos
eve://repos/{repoId}
eve://repos/{repoId}/snapshots
eve://repos/{repoId}/snapshots/{snapshotId}
```

MCP tools:

- `list_repos`
- `list_snapshots`
- `get_snapshot`
- `complete_snapshot`
- `skip_snapshot`
- `checkout_snapshot`

## Security

Version 0 assumes trusted local usage:

- Bind API to localhost.
- Bind MCP to localhost.
- Do not enable remote access by default.
- Treat `.eve/snapshots/*.json` as canonical.
